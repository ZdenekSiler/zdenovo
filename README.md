# zdenovo

A Claude Code project template demonstrating best practices for AI-assisted development: design-first workflow, custom skills, parallel subagents, and modular configuration.

## Getting Started

### Prerequisites

- Python 3.12+
- [uv](https://docs.astral.sh/uv/) — Python package manager
- Git
- Claude Code CLI

### Installation

```bash
git clone <repo-url> zdenovo
cd zdenovo/backend
uv sync
```

### Development

```bash
uv run uvicorn main:app --reload   # Start dev server on :8000
uv run pytest                      # Run tests
uv add <package>                   # Add a dependency
```

## Project Structure

```
zdenovo/
├── CLAUDE.md              # Claude Code instructions
├── .claude/
│   ├── settings.json      # Permissions, hooks
│   ├── rules/             # Modular coding rules
│   └── skills/            # Custom skills: design, implement, simplify
├── docs/
│   ├── architecture.md    # Architecture decisions
│   ├── workflows.md       # Development workflows
│   └── specs/             # Feature specs (generated by /design)
└── src/                   # Source code
```

## Claude Code Integration

This project uses Claude Code with a design-then-implement workflow.

### Workflow

1. **Design first** — run `/design <feature>` to produce a spec in `docs/specs/`
2. **Implement** — run `/implement <feature>` to implement the spec using parallel subagents
3. **Simplify** — run `/simplify` to review and clean up the implementation

### Custom Skills

| Command | Description |
|---------|-------------|
| `/design <feature>` | Researches codebase and writes a feature spec |
| `/implement <feature>` | Implements a spec using parallel subagents |
| `/simplify` | Reviews changed code for quality and efficiency |

### Configuration

- `CLAUDE.md` — main instructions loaded every session
- `.claude/settings.json` — permissions and hooks
- `.claude/rules/` — modular coding conventions

## Contributing

1. Read `CLAUDE.md` and `docs/workflows.md`
2. Use `/design <feature>` before writing any code
3. Run `npm test` before committing
4. Follow the git conventions in `.claude/rules/git.md`
